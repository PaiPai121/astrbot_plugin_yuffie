# Yuffie 自动报警功能使用说明

## 📋 功能概述

Yuffie 现在支持**自动报警功能**！当黄金价格出现异常波动（跳水/暴涨）时，系统会自动推送预警消息给所有订阅用户。

## 🎯 核心特性

- ✅ **实时监控**：7×24 小时监控黄金价格
- ✅ **动态阈值**：基于 ATR（真实波动幅度）智能判断
- ✅ **订阅推送**：只有订阅用户才会收到报警
- ✅ **冷却机制**：同一方向警报 30 分钟内不重复推送
- ✅ **死信队列**：推送失败的消息会暂存并重试

## 📱 用户指令

### 1. 订阅预警

```
/订阅
```

**效果：**
- ✅ 订阅成功！
- 当黄金价格出现异常波动时，系统会自动推送预警消息给您

### 2. 取消订阅

```
/取消订阅
```

**效果：**
- ✅ 取消订阅成功
- 不再接收预警消息

### 3. 查看订阅状态

```
/订阅状态
```

**效果：**
- 显示当前订阅状态
- 显示订阅用户数量和累计推送次数

### 4. 查看订阅统计（管理员）

```
/订阅统计
```

**效果：**
- 显示所有订阅用户列表
- 显示每个用户收到的报警次数

## 📊 报警规则

### 触发条件

```
|价格变化| > k × ATR

其中:
- k = 2.5 (灵敏度系数)
- ATR = 过去 14 周期的真实波动幅度
```

### 冷却时间

- **30 分钟**（同一方向警报）
- 例如：上涨警报触发后，30 分钟内不再推送上涨警报

### 数据源

- **国际金价**：Binance PAXG/USDT（实时）
- **国内金价**：东方财富 Au99.99（实时）

## 🔧 技术实现

### 核心组件

```
plugins/
├── alert_monitor.py          # 预警监控器（后台任务）
├── subscriber_manager.py     # 订阅用户管理（SQLite）
├── subscription_commands.py  # 订阅管理指令
└── analysis_report.py        # 深度分析报告
```

### 工作流程

```
1. 用户发送 /订阅 → 写入数据库
2. AlertMonitor 后台运行 → 实时监控价格
3. 检测到异常波动 → 检查冷却期
4. 获取所有订阅用户 → 逐个推送
5. 记录推送结果 → 失败消息入死信队列
```

### 数据库

```sql
subscribers (订阅用户表)
├── id              # 自增 ID
├── user_id         # 用户 ID（唯一）
├── user_name       # 用户名
├── subscribed_at   # 订阅时间
├── is_active       # 是否激活
├── last_alert_at   # 最后报警时间
└── alert_count     # 收到报警次数
```

## ⚠️ 注意事项

1. **冷却时间**：同一方向警报 30 分钟内不重复推送
2. **休市时间**：周末和节假日不推送（市场休市）
3. **静默期**：开盘前后 30 分钟不推送（过滤跳空假信号）
4. **数据源**：依赖东方财富和 Binance API，网络异常可能影响推送
