# Yuffie 自动报警功能 - 完整使用说明

## 🎯 功能概述

Yuffie 现已支持**自动报警功能**！当黄金价格出现异常波动（跳水/暴涨）时，系统会自动推送预警消息给**所有订阅用户**。

## 📱 用户指令

| 指令 | 功能 | 示例 |
|------|------|------|
| `/订阅` | 订阅黄金跳水预警 | `/订阅` |
| `/取消订阅` | 取消订阅 | `/取消订阅` |
| `/订阅状态` | 查看订阅状态 | `/订阅状态` |
| `/订阅统计` | 查看订阅统计（管理员） | `/订阅统计` |

## 🚀 快速开始

### 步骤 1：订阅预警

在 AstrBot 中发送：
```
/订阅
```

## 📊 报警规则

### 触发条件

```
|价格变化| > k × ATR

其中:
- k = 2.5 (灵敏度系数，可在 config.yaml 配置)
- ATR = 过去 14 周期的真实波动幅度
```

### 冷却机制

- **30 分钟**（同一方向警报）
- 例如：上涨警报触发后，30 分钟内不再推送上涨警报
- 下跌警报独立计算冷却时间

### 休市规则

- **周末**：不推送（市场休市）
- **节假日**：不推送
- **静默期**：开盘前后 30 分钟不推送（过滤跳空假信号）

## 🔧 技术架构

### 核心组件

```
plugins/
├── alert_monitor.py          # 预警监控器（后台任务）
├── subscriber_manager.py     # 订阅用户管理（SQLite）
├── subscription_commands.py  # 订阅管理指令
└── analysis_report.py        # 深度分析报告
```

### 数据库

```sql
subscribers (订阅用户表)
├── id              # 自增 ID
├── user_id         # 用户 ID（唯一）
├── user_name       # 用户名
├── subscribed_at   # 订阅时间
├── is_active       # 是否激活
├── last_alert_at   # 最后报警时间
└── alert_count     # 收到报警次数
```

### 工作流程

```
1. 用户发送 /订阅 → 写入 subscribers 表
2. AlertMonitor 后台运行 → 实时监控价格
3. 检测到异常波动 → 检查冷却期
4. 获取所有订阅用户 → 逐个推送
5. 记录推送结果 → 失败消息入死信队列
6. 后台任务定期重试死信队列
```

## ⚠️ 注意事项

1. **冷却时间**：同一方向警报 30 分钟内不重复推送
2. **休市时间**：周末和节假日不推送（市场休市）
3. **静默期**：开盘前后 30 分钟不推送（过滤跳空假信号）
4. **数据源**：依赖东方财富和 Binance API，网络异常可能影响推送
5. **死信队列**：推送失败的消息会暂存并重试
